    Phone Verification Process

    Phone verification is two steps

    1. First user sends phone number and receives code to his/her phone
    2. After user sends code to api in order phone should be verified

    Send Phone Verification Code

    Example Request

    {
    "phone_number": "+121321312312"
}

    HTTP Request

    POST /api/v3/verifications/send_sms_confirmation

    Will send 6 chars confirmation code to user phone number. You should send jwt token in order this work

    Attributes

    • phone_number String

      Phone number

    Verifiy number by code

    Example Request

    {
       "code": "123343",
       "phone_number": "+121321312312"
}

    HTTP Request

    POST /api/v3/verifications/verify_phone

    will verify phone number with given 6 digit code

    Attributes

    • code String

      123121

    • phone_number String

      +121321312312

    Transfer Reporting

    API for filtering and fetching Transfers.

    Transfer Object

    Attributes

    • id String

      Transfer ID in our DB.

    • user_id Integer

      User ID.

    • amount Integer

      Transfer Amount.

    • currency_object Object

      Currency Object. Example: {“currency”: “USD”,“code”: “USD”,“sign”: “$” }

      Show Hide child attributes

      • currency String

        Currency may be one of the following values: “USD”, “EUR”, “GBP”, “AUD”, “CAD”, “ZAR”.

      • code String

        Currency Code may be one of the following values: “USD”, “EUR”, “GBP”, “AUD”, “CAD”, “ZAR”.

      • sign String

        Currency Sign: “$”, “€” etc..

    • transfer_id Integer

      Tranfer ID in a Payment Service (Stripe, Mangopay).

    • payout_account_id String

      Payout Account ID (empty for Adjustments).

    • bank_name String

      Bank Name of the Payout Account (empty for Adjustments or for CC Payout Accounts ).

    • brand String

      CC Brand of the Payout Account (empty for Adjustments or for Bank Payout Accounts).

    • last4 String

      Last 4 digits of the Payout Account. (empty for Adjustments).

    • beneficiary_name String

      Beneficiary Name.

    • payout_account_type String

      Payout Account Type may be one of the following values: “card”, “bank_account”. (empty for Adjustments).

    • status String

      Transfer Status may be one of the following values: “pending”, “processed”, “failed”.

    • note String

      Transfer Note (optional).

    • transfer_type String

      Transfer Type may be one of the following values: “UsBank”, “Afex”, “Mangopay”, “AfexMangopay”, “Adjustment”.

    • created_at Integer

      When this Transfer was created.

    • updated_at Integer

      When this Transfer was updated.

    Pagination Object

    Attributes

    • current_page Integer

      The current page of the dataset, starting at 1 and not 0.

    • total_pages Integer

      The number of pages in the dataset before pagination, of which this paginated dataset is one.

    • next_page Integer, Null

      Returns the next page number or nil if the current page is the last page.

    • prev_page Integer, Null

      Returns the previous page number or nil if the current page is the first.

    • first_page Boolean

      Returns true if the current page is the first page.

    • last_page Boolean

      Returns true if the current page is the last page.

    • limit_value Integer

      The number of records per page (the final page may have fewer than this number of records).

    • total_count Integer

      The total number of records in the dataset before pagination.

    HTTP Request

    POST /api/v3/transfers

    Retrieve a list of transfers. Transfers may be filtered using the following POST parameters.

    Filtering Request Params

    • filters Object(optional)

      The Filter Object.

      Show Hide child attributes

      • status String(optional)

        The Trasfer Status may be one of the following values: “pending”, “processed”, “failed”.

      • currency String(optional)

        The Trasfer Currency may be one of the following values: “USD”, “EUR”, “GBP”, “AUD”, “CAD”, “ZAR”.

      • transfer_type String(optional)

        The Trasfer StTypeatus may be one of the following values: “local”, “international”, “adjustment”.

      • date Object(optional)

        The Date Filter Object.

        Show Hide child attributes

        • filter_type String

          The Date Filter Type may be one of the following values:“eq”, “gt”, “lt”, “between”.

        • value Object, String

          The Date in format: “YYYY-MM-DD”.

          Show Hide child attributes

          • from String(optional)

            Transfers will be filtered starting from this date. (Use from and to only if filter_type set to “between”)

          • to String(optional)

            Transfers will be filtered up to this amount. (Use from and to only if filter_type set to “between”)

      • amount Object(optional)

        The Amount Filter Object.

        Show Hide child attributes

        • filter_type String

          The Amount Filter Type may be one of the following values:“eq”, “gt”, “lt”, “between”.

        • value Object, String, Integer, Float

          Transfers will be filtered by this amount.

          Show Hide child attributes

          • from String, Integer, Float(optional)

            Transfers will be filtered starting from this amount. (Use from and to only if filter_type set to “between”)

          • to String, Integer, Float(optional)

            Transfers will be filtered up to this amount. (Use from and to only if filter_type set to “between”)

      • page String, Integer(optional)

        The Page number, default is 1.

      • per_page String, Integer(optional)

        The Page size, default is 20.
